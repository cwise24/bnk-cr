CNEInstance

Declares the desired state of a BIG-IP Cloud-Native Edition (CNE) instance. Deploys the underlying data-plane component — the F5 Traffic Management Microkernel (TMM) workload — along with its control-plane components. Configures product type, deployment size, Gateway API enablement, optional modules (firewall ACL, intrusion prevention, intelligent LB, CGNAT, dynamic routing), networking, and registry settings. All other CRs depend on a running CNE instance.

F5BNKGateway

Used with the F5 IPAM Controller (FIC) to automatically allocate VIP addresses for a Gateway. Note: Using BNKGateway in BIG-IP for Kubernetes is optional.

Gateway

Standard Kubernetes Gateway API object. Declares listeners (up to 64 per gateway). HTTPRoute and L4Route attach here via parentRef.

HTTPRoute / L4Route

Routing rules that reference a parent Gateway. HTTPRoute handles L7 path-based traffic; L4Route handles TCP/UDP port-based forwarding.

BNKNetPolicy

Attaches iRule or general network policy at the Gateway level or at an individual Listener. Enables custom traffic manipulation without modifying routes.

iRule Manifest

The TCL iRule definition (inline or via ConfigMap) referenced by a BNKNetPolicy. Contains the actual traffic logic — header rewrites, redirects, persistence, logging, etc.

F5BigFwPolicy

Declares firewall rules (ACLs, zone policies). Applied to the Gateway so all traffic passing through is inspected before reaching routes.

BNKSecPolicy

Higher-level security wrapper that references an F5BigFwPolicy. Provides a Kubernetes-native way to compose and attach security posture to a Gateway.

F5SPKEgress

Enables egress connectivity for internal pods that need access to external networks. Supports three SNAT modes: SNAT none (pod IP used), SNAT automap (TMM external VLAN self-IP, default), or SNAT pool (optionally references an F5SPKSnatpool CR). Must also reference an F5SPKDnscache CR for DNS caching. Applied to the namespace where the TMM pod is configured.

F5SPKSnatpool

An optional pool of routable SNAT IP addresses used when F5SPKEgress is configured with SRC_TRANS_SNATPOOL. By default all IPs are advertised via BGP; use prefixList and routeMaps to restrict advertisement. Supports shared SNAT address mode for large-scale deployments where multiple TMM pods share the same SNAT addresses.

Prometheus

Scrapes real-time inference metrics (GPU utilization, queue depth, token throughput) from LLM backends such as NVIDIA NIM and vLLM. Feeds the f5-analyzer pod with the data it needs to make routing decisions.

f5-analyzer

An optional BNK pod enabled via CNEInstance (intelligentLB: true). Defined by the F5BigAnalyzer CR, it attaches to one or more HTTPRoute or L4Route resources via the applications field. Queries Prometheus metrics and runs a script (built-in LLMLoadMonitor or custom) on a schedule to dynamically adjust route weights for intelligent LLM load balancing. Primarily for DPU installations.
