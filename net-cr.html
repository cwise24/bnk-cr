04 — Egress Traffic Flow (Pod → External) with SNAT & Routes
① Pod Sends Egress Packet
Pod uses TMM's internal VLAN self IP as its default gateway (set by OVN annotation k8s.ovn.org/routing-namespaces). Packet is forwarded to TMM's internal SR-IOV interface.
② TMM: Wildcard Virtual Server Match
F5SPKEgress CR creates a wildcard Virtual Server (0.0.0.0/0 for IPv4, ::/0 for IPv6). Packet matches the egress-ipv4 or egress-ipv6 virtual server.
VS Nameegress-ipv4 / egress-ipv6
dualStackEnabledcreates both VS if true
③ SNAT: Source IP Translation
TMM translates the Pod's cluster IP to a routable SNAT address. Uses either SNAT pool or SNAT automap.
SNAT AutomapUses external VLAN self IP. egressSnatpool left empty.
SNAT PoolF5SPKSnatpool CR supplies per-replica IP list. Referenced by spec.egressSnatpool.
SNAT Pool Scaling: First address list → TMM replica 1, second list → replica 2, etc. SNAT IPs are advertised to BGP unless filtered by prefixList/routeMap.
④ Static Route: External Gateway
F5SPKStaticRoute CR programs TMM's routing table to forward egress packets to the correct upstream gateway. Without a route, TMM cannot deliver to external destinations.
destinationTarget subnet or host IP
gatewayUpstream router IP on ext VLAN
interfaceexternal VLAN interface
typegateway (default)
kind: F5SPKStaticRoute
⑤ Packet Leaves via External VLAN
TMM forwards SNAT'd packet out the external VLAN SR-IOV interface. Source IP is now the SNAT pool address, routable from the external network.
DNS/NAT46 Egress Path (IPv4 Pod → IPv6 Host)
F5SPKDnscache CR
Configures TMM as a DNS caching resolver. Installed first — required by F5SPKEgress.
cacheTypenet-resolver
forwardZonesdomain → upstream DNS IP:port
nameServers.ipAddressMust match dnsNat46PoolIps
F5SPKEgress CR — DNS/NAT46 Mode
Enables IPv4-to-IPv6 translation flow:
dnsNat46Enabledtrue
dnsNat46Ipv4SubnetPrivate IPv4 pool for A-record NAT mappings
dnsNat46PoolIpsUpstream DNS server IP(s)
maxTmmReplicas= number of selfip_v4s entries
maxReservedStaticIpsReserve IPs for manual dSSM entries
egressSnatpoolIPv6 SNAT pool for outgoing traffic
dnsCacheNamenamespace-name of F5SPKDnscache
nat64Enabledtrue = DNS64/NAT64 support
DNS/NAT46 Translation Logic
► Pod DNS query → TMM intercepts
► DNS reply has only AAAA (IPv6) record?
► TMM creates IPv4 NAT entry in dSSM DB
► Returns synthetic A record (from dnsNat46Ipv4Subnet)
► Pod connects to synthetic IPv4
► TMM translates → real IPv6 host using SNAT IPv6 pool
Counters: egress-dns-ipv4 (DNS reqs) | egress-ipv4-nat46 (mappings) | egress-ipv4 (connections)
dSSM Database
Distributed session state manager. Stores DNS46 NAT mapping entries. Required for DNS/NAT46. Survives TMM restarts. Manual entries via mrfdb tool (use reserved IPs only).
Servicef5-dssm-sentinel
Port26379/TCP (Sentinel)
Entry fmtt_dns46<IPv4> → IPv6
05 — Control Plane: SPK Controller → TMM Configuration Push
SPK Controller (spk-controller)
Watches all SPK CRDs in the cluster. Translates CRs into TMM configuration objects. Pushes config to TMM via gRPC. Manages Kubernetes Finalizers to ensure config cleanup on CR deletion.
watchNamespaceProject scope for OVN annotations
gRPC endpoint→ f5-tmm container
FinalizerCR not deleted until backend cleaned

f5-tmm-routing (ZebOS/BGP)
Sidecar container in TMM Pod. Establishes BGP sessions with Top-of-Rack (ToR) switches. Advertises Virtual Server IPs and SNAT Pool IPs so external routers can reach them.

MTU Validation Rules

Max MTU9198 bytes
Default1500 bytes
spec.mtu≤ global.tmmDefaultMTU

⚠ CautionChanging MTU on live VLAN disrupts traffic
⚠ Critical Deployment Rules
• Only one internal VLAN per Project (spec.internal: true)
• VLAN tag in NetworkAttachmentDef must be unset; use CR spec.tag only
• SNAT pool IPs across multiple SPK instances must not overlap
• DNS/NAT46 subnet (dnsNat46Ipv4Subnet) must remain unchanged for lifetime of install
• Manual dSSM entries must use reserved IPs only (maxReservedStaticIps)
• Always delete + reapply F5SPKEgress (not oc apply patch) when modifying
• TMM replicas must equal number of selfip_v4s entries in VLANs
• Install F5SPKDnscache before F5SPKEgress when using DNS/NAT46
